# Framework Ideas

Help shape a `pi-fiction` concept into a useful framework idea.

This is a conversation, not a ticket factory. The goal is to understand the creative experience we want to enable before deciding what to build.

Ideas live in `.wip/ideas/`. The tracker lives at `.wip/README.md`. Completed or abandoned ideas can later move to `.wip/ideas/.archive/` during cleanup.

## Idea Mindset

Focus on the human and AI creative interaction. Ask what the writer is trying to feel, discover, decide, draft, revise, or continue. Avoid turning the idea into a coding task unless implementation details are necessary to preserve the concept.

A good framework idea explains:

- what creative problem or opportunity exists
- how the extension could help
- what the user experience should feel like
- what questions are still open
- what would make the result useful, playful, and not overbearing

## Ideas, Tasks, and Plans

Use this vocabulary in `pi-fiction`:

- **Ideas** shape creative framework concepts before the deliverable is clear.
- **Tasks** describe accepted work with a concrete deliverable.
- **Plans** are implementation plans for tasks, following the normal `pi-code` meaning.

A framework idea can produce one or more tasks once its shape is clear. If the work still needs discovery, keep it as an idea.

## Process

1. **Explore the idea.** Use `$ARGUMENTS` and conversation context as the starting point. If the idea is fuzzy, ask a few useful questions. If it is already clear, proceed.
2. **Find the creative purpose.** Identify the writing moment this idea serves: ideation, character work, worldbuilding, planning, drafting, review, continuity, recovery, momentum, or something else.
3. **Shape the idea.** Draft a concise idea with `Problem`, `Solution`, and optional `Requirements` sections.
4. **Name the idea.** Use `.wip/ideas/{topic_slug}-{issue_number}-{name}.md`.
5. **Update the tracker.** Add or move the idea in `.wip/README.md` under its current progress state.
6. **Report the path.** Summarize the idea and ask what should change.

Do not create an idea file if the user only wants to brainstorm. Create or edit files when the user wants the idea captured.

## Naming

Files use:

```text
.wip/ideas/{topic_slug}-{issue_number}-{name}.md
```

- `topic_slug`: broad area, lowercase and hyphenated, such as `onboarding`, `prompts`, `characters`, `context`, `review`, `extension`, or `branching`.
- `issue_number`: next two-digit number for that topic, such as `01`, `02`, `03`.
- `name`: short hyphenated summary, usually three to six words.

Example:

```text
.wip/ideas/characters-01-discover-pressure-points.md
```

## Progress States

Use simple progress language:

- `spark`: interesting idea, not shaped yet
- `shaping`: actively being discussed or designed
- `ready`: clear enough to become one or more tasks
- `active`: currently being turned into tasks or implementation work
- `done`: finished and ready to archive later
- `parked`: intentionally set aside

## Tracker

`.wip/README.md` is the lightweight workshop board. Keep it useful, not bureaucratic.

Group ideas by progress state:

1. Active
2. Ready
3. Shaping
4. Sparks
5. Parked
6. Done

Each entry should be one line:

```markdown
- [idea-id](ideas/idea-id.md) - Short human-readable title.
```

When changing progress, update both the idea frontmatter and the tracker in the same edit.

When cleaning up finished work, move done ideas to `.wip/ideas/.archive/` and update the Done section to point to the archived path or remove it if the user wants a clean board.

## Idea Template

```markdown
---
status: shaping
topic: characters
---

# Discover character pressure points

## Problem

What is missing, confusing, too rigid, or creatively underpowered for the writer or agent?

## Solution

How could `pi-fiction` help? Describe the desired user experience and creative effect.

## Requirements

Optional. Include only constraints that matter for preserving the experience.
```

## Writing Style

Write for future creative collaborators. Be clear, lively, and practical. Avoid implementation minutiae unless it changes the user experience. Never mention that an idea was generated by AI.
