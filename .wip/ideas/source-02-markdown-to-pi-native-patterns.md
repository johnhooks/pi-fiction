# Markdown concepts to Pi-native patterns

## Problem

The older `../writing` project tried to support fiction through markdown documents, templates, and workflows. Some of those ideas are still valuable, but `pi-fiction` can do more than copy markdown files. It can use skills, commands, tools, generated context, extension behavior, and interactive flows.

We need to recover the useful creative patterns without preserving the old shape by default.

## Solution

Identify the strongest markdown-era patterns and redesign them as Pi-native interactions. Instead of asking whether a document should be ported, ask what writing moment it served and what the harness can now do better.

For branching fiction, this may include story goals, style guidance, character pressure, continuity tracking, issue tracking, drafting mode, review passes, and context packets. The result should be a small set of reusable interaction patterns rather than a large copied template library.

## Requirements

- Treat `../writing` as source material, not a blueprint.
- Preserve creative concepts only when they help a writer or agent move a story forward.
- Prefer interactive flows, skills, commands, tools, and generated context over static templates when they fit better.
- Identify which patterns matter specifically for branching fiction.
- Keep story canon outside the framework repo.
