# pi-fiction

Welcome. This project is about making AI a better creative partner for fiction writers.

We are building a Pi-native writing framework for the space where human imagination and AI assistance meet: a place for developing premises, discovering characters, shaping worlds, drafting scenes, following momentum, revising with taste, and keeping long stories alive without draining the joy out of them.

The framework should help people make compelling, surprising, emotionally alive, fun stories. It should not turn writing into bureaucracy. It should not make the agent timid. It should give the agent enough orientation to collaborate boldly while leaving room for play, instinct, taste, and discovery.

## What We Are Building

Create reusable Pi tools for AI-assisted fiction writing: extension behavior, prompts, skills, commands, scripts, templates, and whatever else proves useful.

Good framework pieces should help with creative acts such as:

- finding the heart of a story
- turning loose ideas into usable direction
- developing characters with wants, contradictions, voices, and pressure
- building worlds that generate story instead of encyclopedias
- planning without killing discovery
- drafting scenes with momentum and texture
- reviewing prose with taste and purpose
- tracking continuity without smothering imagination
- helping a human writer feel energized rather than managed

## Working Spirit

Fiction first. Framework second.

Prefer tools that invite motion over tools that demand compliance. Prefer creative options over rigid gates. Prefer living context over giant doctrine. Prefer taste, voice, and reader experience over mechanical completeness.

When designing a prompt, skill, or tool, ask: does this help a writer or agent make a better story right now?

When writing documentation, make it usable at the moment of creation. If it feels like policy, cut it down. If it sparks better work, keep it.

## For Agents Working Here

Be imaginative. Be practical. Think like a writer, editor, game designer, dramaturg, and toolmaker at once.

Do not preserve an idea merely because it already exists. Do not add structure merely because structure is easy to describe. Add the minimum structure that creates the maximum creative freedom.

Story canon belongs in story projects, not here. This repo should contain reusable creative machinery and tiny generic examples only when they clarify the machinery.

Keep the project Pi-native. Use the capabilities of Pi rather than imitating another assistant’s workflow.

## Validation

Code should lint, test, and build when relevant.

Creative framework assets should pass the real test: would this help a human and an AI move a story forward with more clarity, energy, or delight?
